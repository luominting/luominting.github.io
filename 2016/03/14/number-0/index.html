一、准备开始项目

首先你需要 Node.js 和npm包管理工具
准备好了以上，前往到命令行，我们开始配置项目

克隆项目

现在打开 http://localhost:8080
欢迎浏览一下代码，看看我们是怎么使用webpack 和 npm 脚本来运行应用程序

做些改变

二、渲染Route

作为它的核心, React Router 是一个组件

在我们配置路由之前,这里不会显示任何东西
打开 index.js，然后

  1. 导入 Router 和 Route
  2. 渲染 Router 替换 App

确保你的服务是在运行的，然后访问 http://localhost:8080
你看到的网页上的显示是没有变化的, 但是你可以看到URL上多了哈希部分。
我们用 hashHistory – 通过url的哈希部分来管理 routing history 。
它用额外的
It’s got that extra junk to shim some behavior the browser has natively when using real urls.
We’ll change this to use real urls later and lose the junk, but for now, this works great because it doesn’t require any server-side configuration.

增加页面

创建两个新组建:

  • modules/About.js
  • modules/Repos.js

现在，我们可以将它们耦合到应用程序用各自的路径。

访问 http://localhost:8080/#/about 和 http://localhost:8080/#/repos

三、Navigating with Link

Perhaps the most used component in your app is Link. Its almost identical to the <a/> tag you’re used to except that its aware of the Router
it was rendered in.
也许在你的应用程序中最常用的组件就是Link，这几乎等同于你已经习惯了，只是它知道它在渲染路由器的标签。

在应用程序组件中创建一些导航
